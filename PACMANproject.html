Pac-Man Game Project

Project Overview

This project involves developing a Pac-Man game using various algorithms like Depth-First Search (DFS), A*, and Dijkstra's to manipulate game logic. The project serves as both a practical application of algorithm theory and a demonstration of game development skills.

Technologies Used: Python, Pygame, DFS, Dijkstra's, A* Algorithm, Game Development.

GitHub Repository: View the project on GitHub

Project Description

The Pac-Man game project is a software application developed to simulate the classic Pac-Man arcade game. The project explores the implementation of various search algorithms, such as Depth-First Search (DFS), Dijkstra's, and A*, to control the behavior of PacMan and optimize the gameplay experience.

This project not only showcases game development skills but also demonstrates the practical application of complex algorithms in a gaming environment. The project tracks and compares the performance of each algorithm, intending to explore the performance of different ideas for traversal in Pac-Man.

Features

The Pac-Man game includes several key features:

  • Classic Pac-Man Gameplay: Recreates the classic gameplay experience with Pac-Man navigating through a maze while avoiding ghosts.
  • Algorithm-Driven AI: Uses Depth-First Search (DFS), Dijkstra's, and A* algorithms to control Pac-Man's and the ghosts' behavior, providing a challenge in AI to beat the game.
  • Customizable Levels: Allows for the creation of custom levels with varying difficulty.
  • High Score Tracking: Tracks and displays the highest scores achieved by players.
  • Responsive Controls: Implements responsive keyboard controls for smooth gameplay.

Algorithm Implementation

The Pac-Man game uses the following algorithms to enhance gameplay:

  • Depth-First Search (DFS): Used to generate maze paths and define the initial layout of the game levels.
  • A* Algorithm: Implemented to control ghost movement, making them more intelligent and challenging for the player.
  • Dijkstra's Algorithm: Explored as an alternative method for ghost movement, though observed to be less effective as it often avoids pacgums and demonstrates less optimal behavior.

Each algorithm's performance is tracked and compared to understand the strengths and weaknesses of different traversal methods. Observations include:

  • BFS (Breadth-First Search):
    • BFS to the nearest pacgum
    • BFS to a random spot
    • BFS to powerups then ghosts
    • Observations: "Kinda smart, kinda dumb," sometimes slow, but capable of winning and earning extra lives.
  • A* Algorithm:
    • A* to a random spot
    • A* to powerups then ghosts
    • Observations: Similar success rates as BFS, sometimes smart, sometimes dumb, and occasionally wins extra lives.
  • Dijkstra's Algorithm:
    • Dijkstra to a random spot
    • Observations: "Very dumb," prefers avoiding pacgums, with room for improvement.

Heuristics like Manhattan distance are used in the A* algorithm to determine the best path.

Project Structure

The project is structured as follows:

  • src/: Contains the source code for the game, including the main game loop and AI logic.
  • assets/: Contains all game assets, such as images and sounds used in the game.
  • levels/: Contains level design files that define the layout and difficulty of each level.
  • requirements.txt: Lists the Python dependencies needed to run the game.

The main class structure includes:

  • `game` field: Stores an instance of the game.
  • `setup` method: Performs any pregame calculations.
  • `get_dir` method: Returns the correct direction for the current game state.
