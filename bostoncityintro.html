Low Vision Reader App

Project Overview

This Android app provides an accessible introduction to Boston through two modes: regular and low vision. Users can toggle between two fragments to display city images in different layouts designed to suit both regular and low vision preferences. The project highlights fragment management, accessibility-focused design, and custom UI layouts for optimal readability.

Key Features

  • Two Viewing Modes: Regular and low vision modes provide tailored viewing options for different user needs.
  • Custom Fragments: Two fragments (RegularFragment and LowVisionFragment) display images using unique layouts suited for each mode.
  • Toggle Button: Users can easily switch between modes using a button that loads the appropriate fragment.
  • Accessible UI: Low vision mode uses a vertical layout with larger images for improved readability.

Technical Details

The app is developed using Kotlin in Android Studio. The main activity contains a `ScrollView` with a `LinearLayout` that houses two fragments: RegularFragment and LowVisionFragment. The RegularFragment uses a 2x2 `TableLayout` for image display, while the LowVisionFragment utilizes a vertical `LinearLayout` for larger, stacked images.

Using a custom `showCity(frag: Fragment)` function, the app efficiently switches between fragments to provide different viewing experiences. City data and images are stored in resources, with each image displayed according to the user's selected mode.
